# Framework Catalog

Use this file only to choose the framework. Then read the selected framework file.

## Available frameworks

### `patching-packaging`

- File: [patching-packaging.md](./patching-packaging.md)
- Best for:
  - policy design
  - policy portfolio assessment
  - governance coordination
  - infrastructure and resilience policy
  - path-dependent policy systems that cannot be rebuilt from scratch
- Typical questions:
  - Are current goals and tools aligned?
  - Is this policy system coherent, consistent, and congruent?
  - Should reform rely on wholesale redesign or incremental patching?
  - What can be improved under existing institutional constraints?

### `instrument-gap-scan`

- File: [instrument-gap-scan.md](./instrument-gap-scan.md)
- Best for:
  - literature review
  - research design
  - identifying blind spots in policy instrument analysis
  - diagnosing what an existing policy study still fails to explain
  - building a stronger analytical section in a thesis or report
- Typical questions:
  - What dimensions are missing from this policy analysis?
  - What should be added to make the study more complete?
  - Which variables or mechanisms are under-examined?
  - How can a policy tools study be made more rigorous?

### `policy-mix-recommendation`

- File: [policy-mix-recommendation.md](./policy-mix-recommendation.md)
- Best for:
  - turning diagnosis into recommendations
  - advising government or public agencies
  - designing cross-department policy packages
  - sequencing short-, medium-, and long-term actions
  - assigning lead actors and support actors
- Typical questions:
  - What policy mix should the government adopt?
  - Which departments should do what?
  - How should the recommendations be sequenced?
  - Which tools belong in the short term and which need longer-term reform?

## Current default

If the user is discussing flood governance, drainage policy, climate adaptation, urban resilience, or cross-department policy design, default to `patching-packaging` unless they specify another framework.

If the user is discussing literature gaps, analytical weaknesses, research agenda items, measurement problems, or study design improvements, default to `instrument-gap-scan`.

If the user explicitly wants recommendations to government or departments, or wants an actionable policy package, default to `policy-mix-recommendation`.
