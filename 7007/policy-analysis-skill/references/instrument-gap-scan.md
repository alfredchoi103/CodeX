# Policy Instrument Gap Scan

Based on Giliberto Capano and Michael Howlett, "The Knowns and Unknowns of Policy Instrument Analysis: Policy Tools and the Current Research Agenda on Policy Mixes" (2020).

## When to use

Use this framework when the user does not mainly need a verdict on whether a policy is good or bad, but instead needs to identify:

- what an existing analysis leaves out
- what dimensions a thesis or report should add
- where the literature is still weak
- how a policy instruments study can become more rigorous

This framework is especially useful for:

- literature reviews
- thesis framing
- research gap sections
- methodology design
- upgrading a descriptive policy analysis into an analytical one

## Core idea

The article argues that policy instrument research is rich but uneven. Many studies describe policy tools, but still miss key explanatory dimensions. The main value of this framework is to scan an analysis for blind spots.

## Four gap clusters

### `dynamics`

The study may overlook how tools and mixes evolve over time.

Key issues:

- sequencing
- trajectories
- critical junctures
- layering, drift, conversion
- sectoral and national variation
- sector intersection and convergence
- links to ideas and policy paradigms

### `behavior`

The study may treat policy actors as too simple.

Key issues:

- target behavior
- compliance and non-compliance
- gaming, evasion, free-riding
- decision-makers' actual tool choice behavior
- how tools trigger behavioral mechanisms

### `measurement`

The study may list tools without measuring them properly.

Key issues:

- defining tool types
- measuring tool density
- measuring intensity or coerciveness
- assessing performance and effectiveness
- anticipating volatility, failure, and perverse outcomes

### `implementation`

The study may understate delivery realities.

Key issues:

- government capacity
- multilevel governance
- tool calibration
- procedural tools
- implementation styles

## Diagnostic questions

Ask these before building the table:

1. Does the current analysis explain how the policy mix evolved over time?
2. Does it examine how targets and policy-makers actually behave?
3. Does it define and measure tools in a systematic way?
4. Does it explain how implementation capacity affects outcomes?
5. Which of the four clusters are covered well, partly, or not at all?

## Primary gap-scan table template

| Gap cluster | What to check | Typical weakness | Why it matters | What to add |
| --- | --- | --- | --- | --- |
| Dynamics | Does the study explain sequencing, layering, or historical evolution? | treats the policy mix as static | misses path dependence and reform constraints | add temporal analysis |
| Behavior | Does the study explain target response and tool-choice behavior? | assumes compliance or rational uptake | misses resistance, gaming, or low uptake | add actor-behavior analysis |
| Measurement | Are tools, intensity, and effects clearly defined and measured? | only lists instruments descriptively | cannot compare or evaluate effectiveness | add indicators and coding rules |
| Implementation | Does the study assess capacity, calibration, and multilevel delivery? | assumes implementation is automatic | misses feasibility and delivery gaps | add implementation analysis |

## Secondary research-design table template

Use this when the user wants to improve a thesis, dissertation, or report.

| Research weakness | Symptom in the current draft | Upgrade action | Example for flood governance |
| --- | --- | --- | --- |
| Static analysis | only describes current measures | reconstruct the policy sequence over time | trace how drainage, planning, and warning tools were added |
| Thin actor model | treats the public or firms as passive | analyze incentives, resistance, and compliance | examine developers, property managers, and residents |
| Weak operationalization | names tools without measurement | define categories, intensity, and outcome indicators | code standards, budgets, mandates, and response thresholds |
| Implementation blind spot | ignores who can actually deliver | assess capacity and cross-level coordination | map DSD and cross-department delivery capability |

## Interpretation guidance

### Use this framework to say

- what is missing
- what needs to be added
- why a current analysis is still incomplete
- how to improve analytical rigor

### Do not use this framework alone to say

- whether the policy mix is coherent overall
- whether patching or packaging is the better reform strategy

For those questions, pair it with `patching-packaging`.

## Applied guidance for Hong Kong inland flooding

When using this framework for Hong Kong inland flooding, look for these common blind spots:

| Cluster | Common blind spot in flood studies |
| --- | --- |
| Dynamics | current governance is described without tracing how the mix accumulated over time |
| Behavior | little attention to developers, building managers, transport operators, and household response |
| Measurement | tools are listed but not coded by type, intensity, or effect |
| Implementation | strong emphasis on plans, weak attention to cross-department capacity and calibration |

## Output pattern

When using this framework, structure the answer as:

1. `Framework chosen`
2. `Main analytical gaps`
3. `Gap scan table`
4. `Implications for the user's case`
5. `How to strengthen the analysis`
